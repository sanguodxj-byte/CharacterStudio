# 程序控制表情动画清单

## 说明

本文档整理当前 `CharacterStudio` 中“可由程序直接驱动/切换”的表情动画状态，来源于运行时代码，而不是仅依据资源命名约定。

这里的“程序控制”包括两类：

1. **运行时自动判定的表情状态**：由 Pawn 当前状态、Job、精神状态、需求值等自动推导。
2. **编辑器预览强制覆盖的表情状态**：由预览器手动指定或自动播放序列直接写入。

> 注意：
>
> - 枚举里声明的表情，不代表一定有对应贴图资源；是否最终能看到效果，还取决于 `faceConfig` 是否提供了该表情或分层部件资源。
> - `Blink` 属于内部驱动状态，也可在编辑器预览中被直接指定。
> - 当前文档仅整理“表情状态与程序触发逻辑”，不展开资源导入规范。

---

## 一、当前可用的表情状态总表

当前 [`ExpressionType`](../Source/CharacterStudio/Core/PawnFaceConfig.cs) 枚举包含以下程序可识别状态：

### 1. 基础情绪

- `Neutral`：平静 / 默认回退
- `Happy`：快乐
- `Cheerful`：愉快
- `Gloomy`：阴郁
- `Sad`：悲伤
- `Hopeless`：绝望

### 2. 生理/身体状态

- `Tired`：疲劳
- `Pain`：痛苦
- `Sleeping`：睡眠

### 3. 战斗/危险状态

- `Angry`：愤怒
- `Scared`：恐惧
- `Shock`：震惊
- `WaitCombat`：战斗待机 / 警戒
- `AttackMelee`：近战攻击
- `AttackRanged`：远程攻击

### 4. 日常活动状态

- `Eating`：进食
- `Working`：工作 / 制作
- `Hauling`：搬运
- `Reading`：阅读
- `SocialRelax`：社交放松
- `Lovin`：亲密
- `Strip`：脱装备
- `Goto`：移动中
- `LayDown`：躺下

### 5. 特殊状态

- `Dead`：死亡
- `Blink`：眨眼

## 二、当前“已在运行时代码中实际接入”的自动控制表情

以下状态已在 [`CompPawnSkin.UpdateExpressionState()`](../Source/CharacterStudio/Core/CompPawnSkin.cs) 中明确接入自动判定逻辑，会由程序在运行时主动切换：

| 表情 | 触发条件 | 代码来源 |
|---|---|---|
| `Dead` | Pawn 死亡 | `pawn.Dead` |
| `Pain` | Pawn 倒地 | `pawn.Downed` |
| `Sleeping` | 在床上/休息 | `RestUtility.InBed(pawn)` |
| `Eating` | 正在进食 | `JobDefOf.Ingest` |
| `Lovin` | 正在亲密互动 | `JobDefOf.Lovin` |
| `LayDown` | Job 名匹配 `LayDown` | `job.defName` 判定 |
| `Strip` | 脱装备相关 Job | `JobDefOf.Strip` 或名称匹配 |
| `Hauling` | 搬运/携带相关 Job | `HaulToCell` / `HaulToContainer` / 名称匹配 |
| `Reading` | 阅读/学习相关 Job | 名称匹配 `Read` / `Study` |
| `SocialRelax` | 社交放松 | 名称匹配 `SocialRelax` / `Chat` |
| `Working` | 制作、研究、清洁、建造、种植等工作 | `DoBill` 或多个工作类 Job 名称匹配 |
| `WaitCombat` | 战斗警戒 | 名称匹配 `WaitCombat` |
| `AttackMelee` | 近战攻击 | 名称匹配 `AttackMelee` |
| `AttackRanged` | 远程攻击 | 名称匹配 `AttackStatic` / `Shoot` / `Burst` |
| `Scared` | 惊慌/逃跑类精神状态 | `MentalStateDef` 名称匹配 `Flee` / `Panic` / `WildMan` |
| `Angry` | 攻击性精神状态 | 非恐慌型精神状态分支 |
| `Tired` | 休息值低于 0.15 | `rest < 0.15f` |
| `Hopeless` | 心情低于 0.1 | `mood < 0.1f` |
| `Sad` | 心情低于 0.2 | `mood < 0.2f` |
| `Gloomy` | 心情低于 0.4 | `mood < 0.4f` |
| `Cheerful` | 心情高于 0.9 | `mood > 0.9f` |
| `Happy` | 心情高于 0.8 | `mood > 0.8f` |
| `Neutral` | 以上条件均不满足时回退 | 默认分支 |

### 当前未在 `UpdateExpressionState()` 中自动赋值的枚举值

以下表情虽然存在于 [`ExpressionType`](../Source/CharacterStudio/Core/PawnFaceConfig.cs) 中，但**目前没有在主运行时自动判定函数里直接赋值**：

- `Shock`
- `Goto`

这意味着：

1. 它们仍然可以作为资源配置项存在；
2. 也可以在编辑器预览中作为强制覆盖目标使用（如果 UI/调用方传入）；
3. 但当前普通 Pawn 运行时不会自动切到这两个状态，除非后续增加专门逻辑。

## 三、编辑器预览中可直接强制切换的表情

编辑器预览通过 [`SetPreviewExpressionOverride(...)`](../Source/CharacterStudio/Core/CompPawnSkin.cs) 强制覆盖当前表情。

在预览菜单中，逻辑上会遍历整个 [`ExpressionType`](../Source/CharacterStudio/Core/PawnFaceConfig.cs) 枚举，因此**理论上所有枚举值都可以成为预览目标**，包括：

- `Neutral`
- `Happy`
- `Cheerful`
- `Gloomy`
- `Sad`
- `Hopeless`
- `Tired`
- `Pain`
- `Sleeping`
- `Angry`
- `Scared`
- `Shock`
- `WaitCombat`
- `AttackMelee`
- `AttackRanged`
- `Eating`
- `Working`
- `Hauling`
- `Reading`
- `SocialRelax`
- `Lovin`
- `Strip`
- `Goto`
- `LayDown`
- `Dead`
- `Blink`

是否最终“看起来有效”，取决于：

1. 当前皮肤的 `faceConfig` 是否启用；
2. 是否提供了该表情对应的整脸贴图或分层部件；
3. 是否有相应的通道贴图（嘴、眼睑、眉、情绪覆盖等）。

## 四、当前预览器内置自动播放序列

编辑器顶部 `Auto` 预览按钮当前使用的是一条固定序列，定义于 [`Dialog_SkinEditor`](../Source/CharacterStudio/UI/Dialog_SkinEditor.cs) 中。

### 表情序列

当前 [`PreviewAutoPlayExpressions`](../Source/CharacterStudio/UI/Dialog_SkinEditor.cs) 的顺序为：

1. `Neutral`
2. `Happy`
3. `Neutral`
4. `Angry`
5. `Neutral`
6. `Blink`
7. `Sad`
8. `Neutral`

### 视线方向序列

当前 [`PreviewAutoPlayEyeDirections`](../Source/CharacterStudio/UI/Dialog_SkinEditor.cs) 的顺序为：

1. `Center`
2. `Left`
3. `Center`
4. `Right`

### 自动播放特点

- 由 [`UpdatePreviewAutoPlay()`](../Source/CharacterStudio/UI/Dialog_SkinEditor.Preview.cs) 按时间推进；
- 由 [`ApplyPreviewAutoPlayStep()`](../Source/CharacterStudio/UI/Dialog_SkinEditor.Preview.cs) 写入当前预览表情和眼方向；
- 会启用表情覆盖与眼方向覆盖；
- 会清空嘴型 / 眼睑 / 眉毛 / 情绪覆盖的手动预览覆盖，让这些部件跟随表情自动解析；
- 步进间隔当前支持：`0.50s / 0.75s / 1.00s`。

## 五、程序控制表情对分层通道的联动

即便没有单独为某个嘴型/眉型/眼睑状态写 UI，程序在很多情况下也会根据当前表情自动解析出对应通道状态。

这些映射来自 [`CompPawnSkin`](../Source/CharacterStudio/Core/CompPawnSkin.cs) 内的几组解析函数。

### 1. 嘴型联动

由 `ResolveMouthState(...)` 决定，当前映射为：

- `Happy` / `Cheerful` / `Lovin` / `SocialRelax` → `Smile`
- `Eating` / `AttackMelee` / `AttackRanged` / `Scared` → `Open`
- `Gloomy` / `Sad` / `Hopeless` / `Pain` / `Tired` → `Down`
- `Sleeping` / `LayDown` / `Dead` → `Sleep`
- 其他情况 → `Normal`

### 2. 眼睑联动

由 `ResolveLidState(...)` 决定，当前映射为：

- `Blink` → `Blink`
- `Sleeping` / `Dead` → `Close`
- `Tired` / `Gloomy` / `Sad` / `Hopeless` / `Pain` / `LayDown` → `Half`
- `Happy` / `Cheerful` / `Lovin` → `Happy`
- 其他情况 → `Normal`

### 3. 眉毛联动

由 `ResolveBrowState(...)` 决定，当前映射为：

- `Angry` / `WaitCombat` / `AttackMelee` / `AttackRanged` → `Angry`
- `Gloomy` / `Sad` / `Hopeless` / `Tired` / `Pain` / `Dead` → `Sad`
- `Happy` / `Cheerful` / `Lovin` → `Happy`
- 其他情况 → `Normal`

### 4. 情绪覆盖联动

由 `ResolveEmotionOverlayState(...)` 决定，当前映射为：

- `Lovin` → `Lovin`
- `Gloomy` / `Sad` / `Hopeless` → `Gloomy`
- 其他情况 → `None`

## 六、Blink（眨眼）属于特殊程序动画

`Blink` 与普通状态不同，它除了可以在预览器中被强制指定外，还会在运行时随机触发：

- 逻辑位置：[`UpdateBlinkLogic()`](../Source/CharacterStudio/Core/CompPawnSkin.cs)
- 触发前提：
  - 当前没有表情预览覆盖；
  - 当前表情不是 `Sleeping`；
  - 当前表情不是 `Dead`。
- 触发方式：
  - 每隔一段 Tick 做概率判定；
  - 命中后进入短暂 `Blink` 状态；
  - 结束后请求渲染刷新。

因此，`Blink` 既是：

1. **自动程序动画**；
2. **预览器可手动指定的表情**；
3. **自动播放预览序列中的内置状态**。

## 八、结论

当前系统中，“程序控制表情动画”并不是只有少量几个状态，而是已经具备一整套可扩展的表情枚举与运行时驱动框架。

从“真正已接入程序逻辑”的角度看：

- **自动运行时驱动**：已覆盖绝大多数常用情绪、工作、战斗、休息、死亡状态；
- **编辑器手动预览驱动**：理论上可覆盖全部枚举值；
- **自动播放预览驱动**：当前内置一条轻量演示序列；
- **通道联动驱动**：嘴型、眼睑、眉毛、情绪覆盖会随表情自动变化。

若后续需要继续扩展，最直接的方向是：

1. 给 `Shock`、`Goto` 增加明确的运行时判定；
2. 扩充自动播放序列；
3. 为更多状态补齐分层资源与多帧动画资源；
4. 将当前清单进一步细化为“资源配置建议表 + 文件命名建议表”。
