# 程序控制表情动画 — 架构说明

> 本文档面向**模组制作者和开发者**，解释 CharacterStudio 的程序化面部动画系统如何工作，
> 各层部件的职责边界，以及为什么某些参数看起来"没有效果"。

---

## 一、系统总览

```
游戏状态（心情、战斗、睡眠...）
        │
        ▼
  ExpressionType（表情枚举：Happy / Scared / Sleeping...）
        │
        ▼
  ┌─────────────────────────────────────────────┐
  │           FaceRuntimePolicy                  │
  │  决定使用哪条渲染轨道（World / Portrait）     │
  │  以及 LOD 等级（影响更新频率）                │
  └─────────────┬───────────────────────────────┘
                │
                ▼
  ┌─────────────────────────────────────────────┐
  │       FaceTransformEvaluator（纯计算）        │
  │                                              │
  │  输入：FaceTransformContext + 各部件 Config   │
  │  输出：FaceTransformResult                   │
  │    ├── angle（旋转角度）                      │
  │    ├── offset（XZ偏移）                      │
  │    ├── scale（XYZ缩放）                      │
  │    └── hidden（是否隐藏）                     │
  └─────────────┬───────────────────────────────┘
                │
                ▼
  ┌─────────────────────────────────────────────┐

  │  ScaleFor   → 缩放                           │
  │  RotationFor → 旋转                          │
  └─────────────────────────────────────────────┘
```

## 二、面部图层层级（从底到顶）

| 绘制顺序 | 部件类型 | 职责 | 是否有程序动画 |
|---------|---------|------|-------------|
| 0 | Base | 脸部底色/皮肤 | ❌ |
| 1 | Eye（眼白） | 巩膜，静止不动 | ❌ 仅缩放 |
| 2 | Pupil（瞳孔） | 跟随注视方向移动 | ✅ 位移+缩放 |
| 3 | UpperLid（上眼睑） | 眨眼/表情开合 | ✅ 位移+缩放+旋转 |
| 4 | LowerLid（下眼睑） | 配合上眼睑 | ✅ 位移+缩放+旋转 |
| 5 | Brow（眉毛） | 情绪表达 | ✅ 位移+缩放+旋转 |
| 6 | Mouth（嘴巴） | 表情/说话 | ✅ 位移+缩放+旋转 |
| 7 | ReplacementEye | 眨眼/闭眼时的替换眼 | ✅ 显隐控制 |
| 8 | ReplacementMouth | 张嘴表情的替换嘴 | ✅ 显隐控制 |
| 9 | Blush（腮红） | 害羞/恋爱 | ✅ 脉动+位移 |
| 10 | Tear（眼泪） | 悲伤/痛苦 | ✅ 脉动+位移+旋转 |
| 11 | Sweat（汗滴） | 紧张/战斗 | ✅ 脉动+位移+旋转 |
| 12 | Hair（头发） | 面部范围内的头发 | ⚠️ 微弱慢波 |
| 13 | Overlay | 通用覆盖层 | 跟随目标部件 |

### 关键设计原则

**眼白（Eye）不移动**。现实中巩膜固定在眼眶中，只有瞳孔（Pupil）会随注视方向偏移。
程序动画只对眼白做**缩放变化**（震惊放大、半闭眼缩小等），绝不做位移或旋转。

## 三、波浪系统（让表情"活"起来）

程序动画的核心是两个正弦波，为面部部件提供微妙的呼吸感：

```
primaryWave = sin(time × 2.25 + phase)   // 主波：较快，约 0.36Hz
slowWave    = sin(time × 0.9 + phase×0.5) // 慢波：较慢，约 0.14Hz
```

- **phase** 由节点哈希自动生成，确保每个部件的波浪不同步
- `primaryWave` 通常用于：X偏移、旋转角度、快速脉动
- `slowWave` 通常用于：Z偏移（纵向微动）、缩放呼吸

这两个波贯穿所有面部部件的动画计算，是表情"不僵硬"的关键。

## 四、各部件动画详解

### 4.1 眼白（Eye）— 静止

```
EvaluateEye():
  offset = (0, 0, 0)       ← 永远为零
  angle  = 0               ← 永远为零
  scale  = (1, 1, scaleZ)  ← 仅 Z 轴缩放随表情变化
```

| 状态 | scaleZ 效果 |
|------|-----------|
| 默认 | 1.0（原始大小） |
| 半闭眼（Half） | 0.92（略压缩） |
| 震惊（Shock） | 1.06+（放大） |
| 睡眠（Sleeping） | 0.88（缩小） |
| 快乐柔（HappySoft） | 0.90（微缩） |

### 4.2 瞳孔（Pupil）— 运动核心

瞳孔是面部动画中最活跃的部件：

```
EvaluatePupil():
  offsetX = baseX + gazeOffset + eyeDirection + variant
  offsetZ = slowWave + gazeOffset + variant
  scale   = pupilScaleVariant + expression
  angle   = primaryWave × transformAngleWave
```

**注视方向偏移**（`PupilMotionConfig`）：
- 正面朝向：左/右瞳孔有独立的 `leftPupil_*` / `rightPupil_*` 参数
- 侧面朝向：使用共享的 `side_*` 参数
- `gazeOffset`：来自 World Track 的精确注视偏移（向量长度限制 ≤ 1.0）

**瞳孔缩放变体**（`PupilScaleVariant`）：

| 变体 | 说明 | 典型值 |
|------|------|-------|
| Focus | 聚焦（工作中） | 0.94 |
| SlightlyContracted | 轻微收缩 | 0.88 |
| Contracted | 收缩（强光） | 0.78 |
| Dilated | 扩张（暗处） | 1.12 |
| DilatedMax | 最大扩张（恐惧） | 1.22 |
| ScaredPulse | 恐惧脉动 | 1.16（波动大） |

### 4.3 上眼睑（UpperLid）— 眨眼主角

```
EvaluateUpperLid():
  眨眼时：从 0 平滑移动到 upperLidMoveDown（约 0.004~0.008）
  闭眼时：固定在 upperLidMoveDown 位置
  半闭时：移动较小距离 + slowWave 微动
  快乐时：下移 + 角度旋转（模拟笑眼）
```

**关键参数**：
- `upperLidMoveDown`：上眼睑最大下移量（配置在 `PawnEyeDirectionConfig` 顶层）
- `upperSideBiasX`：左右眼睑的水平偏移（左眼向左、右眼向右）

### 4.4 眉毛（Brow）— 情绪信号

```
EvaluateBrow():
  Angry:  角度下压(-4.5°) + Z上移 + X拉宽(1.04)
  Sad:    角度上扬(+3.25°) + Z下移 + X微宽(1.02)
  Happy:  角度微下(-1.5°) + Z微上 + X宽(1.03)
  Default: 仅 slowWave 微动
```

所有状态都叠加 `primaryWave` 角度波动和 `slowWave` Z 轴微动。

### 4.5 嘴巴（Mouth）

```
EvaluateMouth():
  Smile: 缩放X拉宽(1.06) + 缩放Z压扁(0.94) + slowWave 旋转
  Open:  Z放大(1.14) + primaryWave 旋转
  Down:  角度下弯(-0.75°) + Z缩小(0.90)
  Sleep: Z缩小(0.84) + 静止
  Eating/Shock: 类似 Open 但参数不同
```

### 4.6 情绪覆盖层

| 覆盖层 | 触发条件 | 动画效果 |
|-------|---------|---------|
| Blush（腮红） | Lovin / Happy / Cheerful | X脉动放大 + Z慢波 |
| Tear（眼泪） | Sad / Pain / Gloomy | 脉动 + primaryWave旋转 |
| Sweat（汗滴） | Scared / Shock / Combat | 脉动 + 大幅旋转(2.5°) + X摇摆 |

## 五、motionAmplitude — 用户的"总开关"

每个分层面部部件（`LayeredFacePartConfig`）有一个 `motionAmplitude` 字段：

```xml
<LayeredFacePartConfig>
  <motionAmplitude>0.003</motionAmplitude>
  ...
</LayeredFacePartConfig>
```

**作用**：在程序化变换结果之上，额外叠加一个波浪偏移：
```
finalOffset.x += motionAmplitude × primaryWave
finalOffset.z += motionAmplitude × slowWave
```

**豁免规则**（代码内部硬编码）：
- **瞳孔（Pupil）**：不受 motionAmplitude 影响，偏移完全由 `EvaluatePupil` 控制
- **眼白（Eye）**：不受 motionAmplitude 影响，保持静止

**使用建议**：
- 眉毛、嘴巴、眼睑：0.001 ~ 0.005（微妙浮动）
- 情绪覆盖层：0.002 ~ 0.008（更明显的存在感）
- 头发：0.0005 ~ 0.002（轻微飘动）

## 六、眼睛动画变体（EyeAnimationVariant）

系统根据游戏状态自动选择眼睛的动画风格：

| 变体 | 触发条件 | 瞳孔效果 | 眼白效果 |
|------|---------|---------|---------|
| NeutralOpen | 默认 | 标准偏移 | 无变化 |
| NeutralSoft | 轻微放松 | Z微下 | scaleZ 0.95 |
| NeutralLookDown | 向下看 | Z加大 | scaleZ 0.93 |
| NeutralGlance | 快速瞟视 | X波浪+侧偏 | 无变化 |
| WorkFocusDown | 工作专注 | Z下移 | 无变化 |
| WorkFocusUp | 仰视工作 | Z上移 | 无变化 |
| HappyOpen | 开心睁眼 | Z微上 | 无变化 |
| HappySoft | 开心柔眼 | — | scaleZ 0.90 |
| HappyClosedPeak | 笑眼闭合 | **隐藏** | **隐藏** |
| ShockWide | 震惊大眼 | Z上移 | scaleZ 1.12+ |
| ScaredWide | 恐惧大眼 | X波浪+Z上 | scaleZ 1.08+ |
| ScaredFlinch | 惊缩 | X慢波+侧偏 | scaleZ 0.94 |

## 七、配置文件结构

### 7.1 用户可配置的参数

```xml
<PawnFaceConfig>
  <!-- 眉毛运动 -->
  <browMotion>
    <angryAngleBase>-4.5</angryAngleBase>    <!-- 愤怒角度（度） -->
    <angryOffsetZBase>-0.004</angryOffsetZBase> <!-- 愤怒纵向偏移 -->
    ...
  </browMotion>

  <!-- 嘴巴运动 -->
  <mouthMotion>
    <smileScaleXBase>1.06</smileScaleXBase>  <!-- 微笑X缩放 -->
    ...
  </mouthMotion>

  <!-- 情绪覆盖层运动 -->
  <emotionOverlayMotion>
    <blushPulseBase>1.04</blushPulseBase>    <!-- 腮红基础脉动 -->
    ...
  </emotionOverlayMotion>

  <!-- 眼睛方向/眼睑/瞳孔 -->
  <eyeDirectionConfig>
    <upperLidMoveDown>0.0044</upperLidMoveDown> <!-- 上眼睑下移量 -->
    <lidMotion>...</lidMotion>                    <!-- 眼睑详细参数 -->
    <pupilMotion>...</pupilMotion>                <!-- 瞳孔详细参数 -->
    <!-- eyeMotion 已清空：眼白不做任何运动 -->
  </eyeDirectionConfig>
</PawnFaceConfig>
```

### 7.2 参数单位参考

| 参数类型 | 单位 | 典型范围 | 说明 |
|---------|------|---------|------|
| offset X/Z | 世界单位 | ±0.001 ~ ±0.005 | 1单位 ≈ 屏幕上1像素的几分之一 |
| angle | 度（°） | ±0.5 ~ ±5.0 | 微妙旋转，不超过10° |
| scale | 比率 | 0.78 ~ 1.22 | 1.0=原始大小 |
| motionAmplitude | 世界单位 | 0.001 ~ 0.008 | 叠加波浪幅度 |
| upperLidMoveDown | 世界单位 | 0.002 ~ 0.008 | 眼睑闭合位移 |

## 八、常见问题

### Q: 为什么我改了 eyeMotion 的参数但没有效果？
A: 眼白（Eye）设计上不应有任何运动。所有注视方向偏移由瞳孔（Pupil）层处理。
`EyeMotionConfig` 已被清空，不再暴露任何参数。

### Q: motionAmplitude 设大了会怎样？
A: 部件会出现明显的左右/上下摇摆。对于眉毛和嘴巴建议不超过 0.005，
情绪覆盖层可以稍大到 0.008。瞳孔和眼白不受此参数影响。

### Q: 怎么让角色看起来更"活"？
A: 三个层次：
1. **基础**：确保 Pupil 层有纹理，注视方向功能已启用
2. **进阶**：为 Brow / Mouth 设置合理的 `motionAmplitude`（0.002~0.004）
3. **高级**：微调各表情的 MotionConfig 参数（如愤怒眉毛角度、微笑嘴型缩放）

### Q: 眨眼时为什么眼白会消失？
A: 这是正常的。眨眼时系统自动隐藏 Eye/Pupil 层，显示 ReplacementEye 层。
这是一种"贴图替换"策略，确保闭眼状态有正确的视觉效果。

### Q: 正面和侧面的瞳孔偏移为什么是分开配置的？
A: 因为 RimWorld 等距视角下，正面（South）和侧面（East/West）的眼睛形状差异很大。
正面时左右瞳孔需要独立的 baseX 和方向偏移；侧面时两只眼共用一套侧面参数。
