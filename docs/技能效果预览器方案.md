# 技能效果预览器方案

## 1. 目标

为 [`Dialog_AbilityEditor`](Source/CharacterStudio/UI/Dialog_AbilityEditor.cs) 设计一套“技能效果预览器”，用于在编辑技能时直接观察：

- 技能基础施放形态
- 效果命中后的逻辑反馈
- VFX 的触发时机、目标位置与重复节奏
- 运行时组件对技能流程的影响

本方案当前定位为**设计文档**，优先明确架构、交互与落地顺序，不在本轮直接实现运行时功能。

---

## 2. 现状基础

当前项目已经具备几块可复用能力：

1. 角色预览体系
   - [`DrawPreviewPanel()`](Source/CharacterStudio/UI/Dialog_SkinEditor.Preview.cs:297)
   - [`RefreshPreview()`](Source/CharacterStudio/UI/Dialog_SkinEditor.PreviewLifecycle.cs:73)
   - [`MannequinManager.DrawPreview()`](Source/CharacterStudio/UI/MannequinManager.cs:285)

2. 技能实际执行入口
   - [`CompAbilityEffect_Modular.Apply()`](Source/CharacterStudio/Abilities/CompAbilityEffect_Modular.cs:25)
   - [`CompAbilityEffect_Modular.QueueVisualEffects()`](Source/CharacterStudio/Abilities/CompAbilityEffect_Modular.cs:113)
   - [`CompAbilityEffect_Modular.PlayVfx()`](Source/CharacterStudio/Abilities/CompAbilityEffect_Modular.cs:167)

3. 视觉特效执行工厂
   - [`VisualEffectWorkerFactory.GetWorker()`](Source/CharacterStudio/Abilities/VisualEffectWorker.cs:152)
   - 各类 [`VisualEffectWorker`](Source/CharacterStudio/Abilities/VisualEffectWorker.cs:12) 子类

4. 技能编辑器现有三栏结构
   - 左侧列表 + 中间属性 + 右侧效果面板，主入口位于 [`DoWindowContents()`](Source/CharacterStudio/UI/Dialog_AbilityEditor.cs:222)

这意味着预览器不必从零开始，可优先走“**编辑器内专用预览层 + 运行时逻辑分层复用**”路线。

## 3. 设计原则

### 3.1 非侵入
不直接在编辑器里真实调用完整战斗结算，避免：

- 误伤地图单位
- 真正生成不可控实体
- 修改正式 Pawn 状态
- 预览一次就污染存档运行态

### 3.2 分层预览
预览拆成三层：

1. **几何层**：范围、目标点、方向、半径、弹道示意
2. **表现层**：VFX、贴图特效、触发时序、重复播放
3. **结果层**：文本化显示伤害、治疗、召唤、位移等预期结果

### 3.3 可降级
部分效果不适合在编辑器完整模拟，例如控制、召唤、地形改造。对这些效果应允许：

- 只显示摘要
- 只显示目标格与图标
- 不实际执行游戏逻辑

### 3.4 复用现有 UI 语言
视觉风格对齐：

- [`UIHelper.PanelFillColor`](Source/CharacterStudio/UI/UIHelper.cs:13)
- [`UIHelper.PanelFillSoftColor`](Source/CharacterStudio/UI/UIHelper.cs:13)
- [`UIHelper.HeaderColor`](Source/CharacterStudio/UI/UIHelper.cs:13)
- [`DrawToolbarButton()`](Source/CharacterStudio/UI/Dialog_AbilityEditor.cs:113)

也就是继续采用“深色面板 + 标题条 + 细边框 + 强调线 + tiny 标题”的统一方案。

## 4. 用户侧功能目标

### 4.1 预览器需要回答的问题
用户在编辑一个技能时，最想立即看到的是：

- 这个技能会打到谁？
- 作用范围有多大？
- 是从施法者出发还是从目标点生效？
- VFX 在施法者、目标还是两者都出现？
- 延迟与重复次数是否符合预期？
- 运行时组件会不会改变技能节奏？

### 4.2 MVP 功能
首版建议只做以下能力：

1. 选择预览施法者与目标点
2. 显示技能作用几何
3. 顺序播放 VFX 预览
4. 文本列出逻辑效果摘要
5. 提供“播放一次 / 自动循环 / 重置”控制

### 4.3 暂不纳入首版
以下能力放到后续迭代：

- 真正对地图 Pawn 施加伤害或 Hediff
- 真正生成召唤物
- 真正改变地形
- 真正触发复杂战斗 AI 反应
- 完整 projectile 飞行物物理模拟

## 5. 推荐界面方案

## 5.1 布局位置
建议在 [`Dialog_AbilityEditor`](Source/CharacterStudio/UI/Dialog_AbilityEditor.cs) 采用“右侧标签页扩展”方式，而不是新开独立窗口。

推荐结构：

- 右侧现有标签：效果 / VFX / RC
- 新增第 4 个标签：**预览**

优点：

- 不破坏当前编辑器主工作流
- 用户在改参数后切到预览即可验证
- 与效果、VFX、运行时组件有强关联，认知连续

### 5.2 预览标签页内部布局
建议分成上下两块：

1. **上半区：预览画布**
   - 小型角色/目标示意区
   - 范围圈、目标格、方向箭头、落点标记
   - VFX 叠加播放

2. **下半区：预览控制台**
   - 施法者/目标模式
   - 触发按钮
   - 时间轴摘要
   - 结果日志

### 5.3 顶部工具条建议
工具条可参考 [`DrawPreviewPanel()`](Source/CharacterStudio/UI/Dialog_SkinEditor.Preview.cs:297) 的交互模式，但视觉语言复用 [`DrawToolbarButton()`](Source/CharacterStudio/UI/Dialog_AbilityEditor.cs:113)。

推荐按钮：

- 播放一次
- 自动循环
- 重置
- 切换目标模式
- 切换网格/范围显示
- 速度倍率（0.5x / 1x / 2x）

## 6. 运行架构方案

### 6.1 总体思路
不要让编辑器直接依赖 [`CompAbilityEffect_Modular.Apply()`](Source/CharacterStudio/Abilities/CompAbilityEffect_Modular.cs:25) 做真实执行，而是抽出一层“预览计划”。

建议新增：

- `AbilityPreviewPlan`
- `AbilityPreviewEvent`
- `AbilityPreviewSimulator`
- `AbilityPreviewRenderer`
- `Dialog_AbilityEditor.Preview.cs`

### 6.2 数据流

```text
ModularAbilityDef
   -> AbilityPreviewSimulator.BuildPlan()
   -> AbilityPreviewPlan
   -> AbilityPreviewRenderer.Draw()
   -> 预览画布 / 时间轴 / 日志面板
```

### 6.3 核心对象建议

#### `AbilityPreviewPlan`
表示一次技能预览的静态结果，包含：

- 技能基础元数据
- 目标模式
- 范围半径
- 预测命中点
- 时间序列事件列表
- 逻辑效果摘要列表

#### `AbilityPreviewEvent`
表示时间轴上的一个事件：

- `timeOffsetTicks`
- `eventType`
- `label`
- `worldPosition`
- `radius`
- `color`
- `iconPath`
- `linkedEffectIndex`
- `linkedVfxIndex`

#### `AbilityPreviewSimulator`
职责：把 [`ModularAbilityDef`](Source/CharacterStudio/Abilities/ModularAbilityDef.cs:56) 转换成只读预览计划。

建议方法：

- `BuildPlan(ModularAbilityDef ability, AbilityPreviewContext context)`
- `SummarizeEffects(...)`
- `ExpandVisualEffects(...)`
- `ExpandRuntimeComponents(...)`

#### `AbilityPreviewRenderer`
职责：渲染计划，不负责业务推导。

建议方法：

- `DrawCanvas(Rect rect, AbilityPreviewPlan plan, float now)`
- `DrawTimeline(Rect rect, AbilityPreviewPlan plan, float now)`
- `DrawResultLog(Rect rect, AbilityPreviewPlan plan)`

## 7. 预览上下文设计

建议新增 `AbilityPreviewContext`，用于描述本次预览环境：

- `casterPosition`
- `targetPosition`
- `casterRotation`
- `hasValidTarget`
- `previewMapSize`
- `selectedPreviewPawnKind`
- `playbackSpeed`

这样可以在不接触真实地图状态的情况下，稳定生成预览计划。

## 8. 三层预览细化

## 8.1 几何层
根据载体类型显示基础作用轮廓：

### Target
- 施法者到目标的连线
- 目标落点高亮
- 显示射程上限

### SelfCast
- 仅高亮施法者
- 若有半径，则显示自身圆形范围

### AOE
- 显示范围圈
- 若中心来自目标点，则圈定目标区域

### Projectile
- 显示起点到目标的弹道线
- 首版只做直线示意，不做真实抛物线模拟

## 8.2 表现层
把 [`CompAbilityEffect_Modular.QueueVisualEffects()`](Source/CharacterStudio/Abilities/CompAbilityEffect_Modular.cs:113) 的时序规则转成预览事件：

- `delayTicks`
- `repeatCount`
- `repeatIntervalTicks`
- `trigger`
- `target`
- `sourceMode`

对于 [`VisualEffectWorkerFactory`](Source/CharacterStudio/Abilities/VisualEffectWorker.cs:139) 中的内建类型，可直接映射为编辑器内可视标记：

- DustPuff -> 灰色扩散圈
- MicroSparks -> 黄色闪点
- LightningGlow -> 蓝白脉冲圈
- FireGlow -> 橙红发光圈
- Smoke -> 半透明烟雾层
- ExplosionEffect -> 组合脉冲

对于 `CustomTexture`：

- 首版优先显示贴图缩略占位
- 次版再考虑直接复用贴图加载绘制

## 8.3 结果层
不真实改动世界，只输出“预测结果摘要”：

- 伤害：`Damage 30 (Blunt)`
- 治疗：`Heal 20`
- Hediff：`Add Hediff: Burn`
- 召唤：`Summon PawnKind: X`
- 位移：`Teleport target/caster`
- 控制：`Apply control for N ticks`
- 地形：`Terraform -> Terrain/Thing`

## 9. 与运行时逻辑的关系

### 9.1 不直接复用完整执行
[`CompAbilityEffect_Modular.Apply()`](Source/CharacterStudio/Abilities/CompAbilityEffect_Modular.cs:25) 当前直接进入真实效果执行，这对于编辑器预览来说太重。

因此建议：

- **运行时执行逻辑** 保持不动
- 新增 **预览解释逻辑**，只读取配置，不落地副作用

### 9.2 可以复用的部分
可复用：

- VFX 时序规则
- 目标解析规则
- 类型标签文本
- 部分贴图加载/颜色映射

不建议直接复用：

- 真实伤害施加
- 地图生成实体
- 真实消息弹窗
- 对 `CompPawnSkin` 的运行态写入

### 9.3 后续可抽象的公共层
长期可把运行时与预览同时依赖的解释部分再抽一层：

- `AbilityExecutionDescriptorBuilder`
- `AbilityVisualEventBuilder`

这样可避免“运行时一套规则、预览器另一套规则”逐渐漂移。

## 10. UI 交互方案

### 10.1 播放控制
预览标签建议维护这些状态：

- `previewPlaying`
- `previewLoop`
- `previewTimeTicks`
- `previewDurationTicks`
- `previewSpeed`
- `previewTargetMode`

### 10.2 常用操作

#### 播放一次
- 从 0 Tick 重置
- 按时间轴推进
- 播放到末尾停止

#### 自动循环
- 到末尾后自动重置
- 方便调延迟、重复次数、节奏感

#### 重置
- 归零时间轴
- 清除临时高亮

#### 目标模式切换
- 无目标
- 单目标近距
- 远距离目标
- 群体中心目标

### 10.3 日志面板
建议在底部增加简易日志流：

- `[0] Cast Start`
- `[10] VFX: LightningGlow @ Target`
- `[10] Effect: Damage 30`
- `[22] VFX: Smoke repeat #2`

这样用户不用盯着动画，也能检查时序。

## 12. 分阶段实施建议

### 阶段 1：静态预览 MVP
目标：最快形成可见反馈。

内容：

- 新增“预览”标签
- 画布显示施法者、目标、范围圈、连线
- 底部显示效果摘要
- 支持 VFX 时间轴点位显示
- 支持播放/循环/重置

### 阶段 2：VFX 动画化
目标：让视觉特效节奏更直观。

内容：

- 内建 VFX 的脉冲动画
- 自定义贴图缩略绘制
- 延迟与重复可视化
- 简单颜色主题区分不同事件类型

### 阶段 3：高级运行时组件可视化
目标：覆盖复杂技能节奏。

内容：

- Q 连段窗口显示
- R 叠层/二段就绪提示
- 运行时组件说明卡片
- 触发条件链路展示

### 阶段 4：半真实沙盒预览
目标：在独立安全上下文中接近真实效果。

内容：

- 独立预览 Pawn / 假目标
- 不写正式地图，仅写临时容器
- 可选高级模式执行部分轻量 runtime 逻辑

## 13. 风险与注意事项

### 13.1 风险：预览与真实效果不一致
解决：

- 首版明确标注“预览为编辑器近似表现”
- 把公共解释规则逐步抽离共享

### 13.2 风险：UI 过重
解决：

- 预览标签默认不自动循环
- 只有切到预览标签时才推进时间轴
- 贴图与事件缓存做轻量化

### 13.3 风险：代码分散后难维护
解决：

- 预览逻辑集中在 `Abilities/Preview/` 目录
- [`Dialog_AbilityEditor`](Source/CharacterStudio/UI/Dialog_AbilityEditor.cs) 只保留状态与布局入口

### 13.4 风险：CustomTexture 外部路径加载失败
解决：

- 复用已有外部贴图解析能力，例如 [`RuntimeAssetLoader.ExternalTextureExists()`](Source/CharacterStudio/Rendering/RuntimeAssetLoader.cs:202)
- 预览器对加载失败贴图降级为占位框 + 日志提示

---

## 14. 与本轮 UI 统一化的结合方式

预览器新增后，应直接遵循当前技能编辑器已开始采用的统一风格：

- 外层卡片壳
- 软色标题条
- 底部强调线
- 工具按钮统一走 [`DrawToolbarButton()`](Source/CharacterStudio/UI/Dialog_AbilityEditor.cs:113) 或同风格抽象
- 信息提示走类似 `InfoBanner` 的轻卡片

避免再次出现旧式 `Widgets.DrawMenuSection(...)` 与默认按钮混用问题。

---

## 15. 最终建议

结论：技能效果预览器应采用“**编辑器内预览标签 + 独立预览计划模拟器 + 分层可视化**”方案。

优先级最高的实现路线是：

1. 先在技能编辑器中加入“预览”标签
2. 先做静态几何 + 时间轴 + 结果摘要
3. 再逐步把 VFX 动画化
4. 最后再考虑半真实 runtime 沙盒预览

这样可以在较低风险下，快速为技能编辑器提供最有价值的反馈闭环，同时与当前 CharacterStudio 的 UI 统一化方向保持一致。
