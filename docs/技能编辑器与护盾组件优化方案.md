# 技能编辑器与护盾组件优化方案

## 1. 现状分析

通过对比《CharacterStudio》当前的技能实现和参考模组（Axolotl，Id:3292351432）的技能实现，我们发现了以下差异与优化空间：

### 1.1 参考模组 (Axolotl) 实现特点
- **高度定制化的组件**：参考模组使用了大量继承自 `HediffComp` 或 `ThingComp` 的自定义逻辑（如 `HediffCompLotlQiShield` 和 `CompAxolotlEnergy`）。
- **护盾视觉渲染**：护盾的视觉效果并非通过在地图上生成 `Thing` 来实现，而是在组件的 `PostDraw()` 方法中，通过 `Graphics.DrawMesh` 配合 `Material` 和 `Matrix4x4` 直接在 Pawn 的 `DrawPos` 处动态绘制，随人物平滑移动。
- **技能多态性**：它拆分了诸如 Fire, Rock, Thunder, Water 等不同的底层 Verb 和 Effect 结构。

### 1.2 《CharacterStudio》当前实现痛点
- **视觉附着物表现生硬**：之前的 `AttachedShieldVisual` 组件通过 `GenSpawn.Spawn` 生成 `Thing` 并每帧同步整数 `Position`。由于 `Position` 只能精确到网格（Cell），导致在人物移动时护盾出现严重的视觉抖动。此问题现已通过重构为在 `CompPawnSkin.PostDraw()` 中直接读取 `Graphic` 进行平滑渲染而解决。
- **编辑器 UI 耦合度高**：在 `Dialog_AbilityEditor.Panels.Shared.cs` 中，每新增一种 `AbilityRuntimeComponentType`，都需要手动编写大量硬编码的 `Widgets.Label` 和 `UIHelper.TextFieldNumeric`。这使得 UI 代码冗长且难以维护。
- **配置数据分散**：`ShieldAbsorb`（护盾吸收数值）与 `AttachedShieldVisual`（护盾视觉附着）目前作为两个独立的枚举类型与组件分别配置，容易导致持续时间等参数出现不一致。

## 2. 优化方案设计

### 2.1 架构层优化：解耦与反射驱动的编辑器
**目标**：消除编辑器代码中的巨型 `switch-case` 或 `if-else` 分支，实现数据驱动的 UI 生成。

**实施步骤**：
1. **引入特性（Attribute）标注**：为 `AbilityRuntimeComponentConfig` 中的字段添加类似 `[EditorField("护盾最大吸收量", Min=1f, Max=99999f)]` 的自定义特性。
2. **构建通用 PropertyDrawer**：在 `Dialog_AbilityEditor` 中实现一个通过反射读取这些特性的渲染器，自动生成对应的文本框或复选框。
3. **优势**：未来新增诸如“反伤护盾”、“引爆护盾”等机制时，只需在配置类中添加字段，UI 便会自动适配。

### 2.2 逻辑层优化：合并机制与表现
**目标**：统一护盾的逻辑与视觉，提供“开箱即用”的完整护盾体系。

**实施步骤**：
1. **聚合配置**：将 `AbilityRuntimeComponentType.ShieldAbsorb` 与 `AttachedShieldVisual` 整合为一个全新的 `ShieldConfig` 类。
2. **联动缩放**：使护盾视觉的 `Scale` 和 `Alpha` 能够与护盾的剩余血量（`shieldRemainingDamage`）挂钩。例如，护盾被击中时视觉效果会闪烁，剩余血量降低时透明度变弱。
3. **支持多种视觉类型**：允许护盾不仅能使用 `Graphic_Single`，还能支持 `Mote` 效果或粒子的动态生成。

### 2.3 渲染层持续优化：双轨管线结合
**目标**：确保护盾与原版渲染管线（PawnRenderTree）和特效层的兼容。

**实施步骤**：
1. **渲染层级分离**：在当前的 `PostDraw` 绘制之上，进一步将护盾绘制插入到 `AltitudeLayer.MoteOverhead` 或特定的 `drawOrder` 中，确保它不会遮挡 UI 或重要的状态提示。
2. **材质复用缓存**：对生成的护盾 `Material` 实例（如使用 `ShaderDatabase.Transparent` 的材质）进行对象池缓存，避免每次重新 `MatFrom` 时带来的微小性能开销或内存碎片。

## 3. 总结
当前已将 `AttachedShieldVisual` 的生成与赋值机制修正为类原版 `MoteAttached` 的平滑跟随机制（基于 `PostDraw` 注入）。
通过上述的组件重构与编辑器解耦方案，将大幅提升《CharacterStudio》技能模块的可扩展性，使其不仅能媲美甚至能超越那些具有深度硬编码逻辑的独立模组。
